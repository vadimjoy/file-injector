# Журнал релизов

> **Language / Язык:** [English](docs/en/RELEASE_NOTES.md) | [Русский](docs/ru/RELEASE_NOTES.md)

## 0.0.1

- Запуск библиотеки вокруг концепции «Deterministic UI Lego» обозначив новую AI-ориентированную архитектуру и пайплайн сборки.
- `SPEC.md` описывает паттерн контекстных модификаторов: поля ввода используют явные классы `.ui-input--icon-left/right/search/readonly`, а оркестраторы (Autocomplete/Datepicker) добавляют только вспомогательные классы (`.ui-autocomplete__input`, `.ui-datepicker__input`) вместо мутаций дочерних атомов.
- CSS компонентов приведён в соответствие с контрактом: из `src/css/components/input.css`, `src/css/components/autocomplete.css` и `src/css/components/datepicker.css` убраны селекторы родителей, поэтому все состояния объявляются на самом элементе.
- Расширена документация (`docs/MASTER_PLAN.md`), чтобы описать многофазную дорожную карту (аудит связностей, контекстные модификаторы, машинно-читаемые спецификации) и дать общий контекст для контрибьюторов и AI-агентов.

## 0.0.2

- Добавлены `scripts/build-modules.js` и npm-скрипт `build:modules`, собирающие отдельные CSS-входы (`tokens.css`, `base.css`, `states.css` и файлы компонентов) напрямую в `dist/` для потребителей, которым нужна файловая tree-shaking.
- Обновлены `.gitignore` и структура репозитория: собранные артефакты больше не хранятся в Git, релизы формируются только через пайплайн сборки.
- Переписан слайдер (`src/css/components/slider.css`): введён типизированный `@property --_fill`, градиентные треки, плавные переходы у ползунка, поддержка Firefox `::-moz-range-progress` и явные стили для disabled. Демо теперь задают `style="--_fill:…"`, чтобы интерфейс отражал CSS-контракт (`src/demos/slider.html`).
- Обновлены демо тумблеров (`src/demos/toggle.html`): лучше показаны включённые/отключённые состояния и современный текст.

## 0.0.3

- Версия пакета повышена до `0.0.3`, а список публикуемых файлов сокращён до готовых артефактов из `dist/` (см. `package.json#files`), что делает установку легче и согласованной с `exports`.
- `package-lock.json` синхронизирован с тем же номером, чтобы потребители получали единообразные метаданные при установках и в CI.

## 0.0.4

- **AI-контекст**: `docs/AI_CONTEXT.md` теперь оформлен как самодостаточный системный промпт (инварианты, реестр компонентов, правила композиции, антипаттерны, справочник токенов, таблицы размеров), который агент подгружает перед генерацией вёрстки.
- **Архитектурные решения**: зафиксированы ADR-0002 (паттерн контекстных модификаторов) и ADR-0003 (нейминг AI-токенов) с обновлённым индексом в `docs/adr/README.md`, чтобы решения оставались пересматриваемыми.
- **Обновление дорожной карты**: `docs/MASTER_PLAN.md` описывает вехи Фазы 2 (стандартизация контекста, система ADR, namespace AI-токенов), демонстрируя, как развивается стратегия «детерминированного Lego».
- **AI-токены компонентов**: в `src/css/tokens.css` добавлен второй слой переменных `--ai-[component]-*`, которые по умолчанию ссылаются на глобальные `--ui-*` и позволяют настраивать отдельные компоненты без побочных каскадов.
- **Переподключение компонентов**: все стили в `src/css/components/` переведены на использование новых `--ai-*` токенов и явных контекстных модификаторов, убирая зависимость от селекторов-родителей ещё до релиза Фазы 2.

## 0.0.5

- **Theme Mapper CLI**: добавлен `scripts/theme-map.js` с поддержкой трёх форматов JSON (родной, W3C Design Tokens, Figma Tokens) и 28 unit-тестов (`npm test`). Теперь достаточно `npm run theme-map -- ./theme.json -o dist/themes/custom.css`, чтобы получить готовый CSS с `@layer ai-kit.themes`.
- **Пять пресетов**: `src/themes/*.json` → `dist/themes/{default,dark,midnight,corporate,warm}.css`, новый скрипт `build:themes` включён в основной пайплайн, а пакет экспортирует `ai-css-kit/themes/*` для точечных импортов.
- **Каcкадные слои**: все токены, базовые стили и компоненты обёрнуты в `@layer ai-kit.tokens/base/components`, а темы — в `ai-kit.themes`, поэтому preset-файлы всегда побеждают без `!important`.
- **Документация**: появился `docs/theming.md`, обновлены `README.md`, `SPEC.md`, `docs/AI_CONTEXT.md` и дорожная карта (Фаза 3 отмечена выполненной). ADR-0004 фиксирует принятое решение.
- **Демо и витрина**: `index.html` получила селектор пресетов, `showcase-page.js` рассылает выбранный theme во все iframe, а `demo-theme.js` поддерживает все значения `data-theme`. Общие CSS демонстраций импортируют готовые preset-файлы, чтобы показать реальное поведение темы.
