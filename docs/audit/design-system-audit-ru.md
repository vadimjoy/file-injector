## 🔴 Критические проблемы

### C-1 · Сборка `dist/components/` охватывает только 27% компонентов

**Файлы:** `dist/components/` vs `src/css/components/`  
**Риск:** Критический — нарушает заявленный контракт модульного импорта

Из 55 компонентов в `src/css/components/` в `dist/components/` опубликованы только 15. Пользователи, подключающие отдельный файл (как описано в SPEC: `@import 'ai-css-kit/components/button.css'`), не получают доступа к 40 компонентам.

**Недостающие в dist:**
`accordion`, `alert`, `avatar`, `banner`, `breadcrumb`, `callout`, `chart-legend`, `chat`, `chip`, `color-swatch`, `divider`, `dot`, `dropdown`, `empty-state`, `feature-item`, `feed`, `footer`, `form-row`, `icon-tile`, `kbd`, `logo-cloud`, `modal`, `navbar`, `notification`, `pagination`, `pricing-card`, `rating`, `search`, `section-header`, `sidebar`, `skeleton`, `spinner`, `stat-card`, `status`, `steps`, `table`, `tabs`, `tag`, `timeline`, `typography`

**Рекомендация:** Скрипт сборки (`scripts/build-modules.js`) нужно распространить на все файлы в `src/css/components/`. Убедиться, что версия пакета и CI отражают полный список компонентов.

### H-3 · `--ui-color-warning-hover` используется без определения

**Файл:** `src/css/tokens.css:456`, компонент `rating`  
**Риск:** Высокий — молчаливый фолбэк скрывает отсутствующий токен

```css
--ai-rating-hover-color: var(--ui-color-warning-hover, #d97706);
```

Токен `--ui-color-warning-hover` нигде не определён (ни в light-теме, ни в dark-теме). Используется захардкоженный fallback `#d97706`, который совпадает с `--ui-color-warning` для светлой темы, но в тёмной теме `--ui-color-warning` меняется на `#fbbf24`, а hover-состояние остаётся некорректным.

**Рекомендация:**
```css
/* В блоке :root (light) */
--ui-color-warning-hover: #b45309;

/* В блоке [data-theme="dark"] */
--ui-color-warning-hover: #d97706;
```

## 🟡 Средний приоритет

### M-1 · `border-radius: 50%` не использует `--ui-radius-full`

**Файлы:** `badge.css`, `avatar.css`, `chip.css`, `toggle.css`, `slider.css`, `steps.css`, `spinner.css`  
**Риск:** Средний — нарушает единый источник истины для радиусов

Жёстко заданные `border-radius: 50%` встречаются в 7 компонентах (в контекстах «полного круга»). `--ui-radius-full: 9999px` семантически эквивалентен и уже используется для badges, но не применяется к внутренним элементам (индикаторам, ручкам ползунка, аватарам).

**Рекомендация:** Заменить `border-radius: 50%` на `var(--ui-radius-full)` там, где элемент должен быть кругом (dot, thumb, spinner circle, avatar). Оставить `50%` только там, где это сознательное поведение без токена (редко).

### M-2 · `footer.css` содержит захардкоженные цвета темы

**Файл:** `src/css/components/footer.css:131-154`  
**Риск:** Средний — значения не меняются при смене темы через JSON

```css
[data-theme="dark"] .ui-footer--dark {
  --ai-footer-bg: #111827;                    /* ← хардкод */
  --ai-footer-border: rgba(255, 255, 255, 0.08); /* ← хардкод */
}
[data-theme="light"] .ui-footer--light {
  --ai-footer-bg: #f8fafc;                    /* ← хардкод */
  --ai-footer-border: rgba(0, 0, 0, 0.08);    /* ← хардкод */
}
```

**Рекомендация:** Заменить на ссылки на глобальные токены: `var(--ui-color-bg-subtle)`, `var(--ui-color-border)` и их тёмные переопределения. Тема-пресет сделает остальное.

### M-5 · `chart-legend` использует класс `.ui-legend`, а не `.ui-chart-legend`

**Файл:** `src/css/components/chart-legend.css`  
**Риск:** Средний — несоответствие имени файла и базового класса

Файл называется `chart-legend.css`, но базовый класс — `.ui-legend`, а не `.ui-chart-legend`. Это ломает конвенцию `имя-файла → базовый-класс` и может вызвать путаницу при именовании токенов.

**Рекомендация:** Либо переименовать класс в `.ui-chart-legend` (мажорный бряк), либо переименовать файл в `legend.css` и обновить имя компонента в документации.

### M-6 · Размеры chip и tag полностью захардкожены, нет токенов

**Файлы:** `src/css/components/chip.css`, `src/css/components/tag.css`  
**Риск:** Средний — нельзя настроить через тему

Отступы, размеры иконок и шрифты для вариантов `--sm`/`--md`/`--lg` заданы голыми пикселями без CSS-переменных:

```css
/* chip.css */
.ui-chip--sm { padding: 4px 10px; font-size: 12px; }
.ui-chip__icon { width: 14px; height: 14px; }

/* tag.css */
.ui-tag--sm { padding: 2px 7px; }
.ui-tag__icon { width: 12px; height: 12px; }
```

**Рекомендация:** Добавить компонентные токены `--ai-chip-padding-*`, `--ai-chip-icon-size`, `--ai-tag-padding-*`, `--ai-tag-icon-size` в `tokens.css`.

## Сильные стороны системы

Аудит выявил не только проблемы — система построена на прочном фундаменте:

- **Двухуровневая токен-архитектура** (`--ui-*` → `--ai-*`) — грамотное разделение глобального и компонентного уровней, чёткий путь переопределения
- **CSS Cascade Layers** — `@layer ai-kit.tokens, ai-kit.base, ai-kit.components, ai-kit.themes` исключают конфликты приоритетов без `!important`
- **Автоматические линтеры** — 4 скрипта полностью проходят: цвета, атомарные уровни, связность, чистота демо
- **Пресеты тем** — 5 готовых тем с Theme Mapper CLI (JSON → CSS) — сильное конкурентное преимущество
- **Atomic Decoupling** (ADR-0001) — компоненты не ссылаются друг на друга через структурные селекторы
- **Полные демо-файлы** — 54 HTML-демо для всех компонентов, включая недокументированные
