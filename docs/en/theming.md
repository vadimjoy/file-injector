# AI CSS Kit — Theming Guide

> **Language:** English | [Русский](../ru/theming.md)

> **Version:** v0.6.0  
> **Related ADR:** [ADR-0003 AI Token Naming](adr/0003-ai-token-naming.md)

## Overview

AI CSS Kit uses a two-level token architecture and CSS `@layer` to make theming
deterministic: a theme file always wins over component defaults without
`!important`.

```
@layer ai-kit.tokens    ← Global --ui-* defaults + dark override
@layer ai-kit.base      ← Reset, .ui-field, .ui-label, …
@layer ai-kit.components ← All 15 component styles (--ai-* tokens)
@layer ai-kit.themes    ← Theme presets / custom themes (highest priority)
```

Any rule placed in `ai-kit.themes` automatically takes precedence over all
component styles, regardless of CSS specificity.

## Using a Preset Theme

Five presets ship in `dist/themes/`:

| File | Description |
|------|-------------|
| `dist/themes/default.css` | Light theme (mirrors built-in `:root` values) |
| `dist/themes/dark.css` | Dark theme (mirrors built-in `[data-theme="dark"]`) |
| `dist/themes/midnight.css` | Deep dark with violet accent, monospace font |
| `dist/themes/corporate.css` | Neutral blue, square corners, system sans-serif |
| `dist/themes/warm.css` | Warm amber tones, generous rounding, serif font |

### 1. Import the theme after the main bundle

```html
<link rel="stylesheet" href="dist/ai-css-kit.css" />
<link rel="stylesheet" href="dist/themes/midnight.css" />
```

### 2. Activate via `data-theme` attribute

```html
<html data-theme="midnight">
  ...
</html>
```

Switch themes at runtime with one line of JavaScript:

```js
document.documentElement.setAttribute('data-theme', 'corporate');
```

## Creating a Custom Theme

### Native format (recommended)

Create a JSON file with a `meta` block and a flat `tokens` object:

```json
{
  "meta": {
    "name": "Ocean Pro",
    "version": "1.0.0",
    "author": "designer@example.com"
  },
  "tokens": {
    "color.primary":       "#0ea5e9",
    "color.primary.hover": "#0284c7",
    "color.bg":            "#f0f9ff",
    "radius.md":           "10px",
    "font.family":         "'Nunito', sans-serif"
  }
}
```

### Token key → CSS variable mapping

| Token key pattern | CSS variable | Example |
|-------------------|--------------|---------|
| `color.*`         | `--ui-color-*` | `color.primary` → `--ui-color-primary` |
| `radius.*`        | `--ui-radius-*` | `radius.md` → `--ui-radius-md` |
| `font.*`          | `--ui-font-*` | `font.family` → `--ui-font-family` |
| `shadow.*`        | `--ui-shadow-*` | `shadow.sm` → `--ui-shadow-sm` |
| `button.*`        | `--ai-button-*` | `button.bg` → `--ai-button-bg` |
| `input.*`         | `--ai-input-*` | `input.border` → `--ai-input-border` |
| `card.*`          | `--ai-card-*` | `card.radius` → `--ai-card-radius` |
| `--custom-var`    | verbatim | `--ui-line-height` → `--ui-line-height` |

Component names that map to `--ai-*`: `button`, `input`, `textarea`, `select`,
`card`, `badge`, `progress`, `tooltip`, `checkbox`, `radio`, `toggle`,
`slider`, `file-upload`, `autocomplete`, `calendar`.

All other token keys map to `--ui-*`.

### Generate the CSS

```bash
node scripts/theme-map.js ./my-theme.json -o ./dist/themes/ocean-pro.css
```

Or using the npm script alias:

```bash
npm run theme-map -- ./my-theme.json -o ./dist/themes/ocean-pro.css
```

The generated CSS:

```css
/* Generated by ai-css-kit theme-map — do not edit */
/* Theme:   Ocean Pro */
/* Version: 1.0.0 */

@layer ai-kit.themes {
  [data-theme="ocean-pro"] {
    --ui-color-primary: #0ea5e9;
    --ui-color-primary-hover: #0284c7;
    --ui-color-bg: #f0f9ff;
    --ui-radius-md: 10px;
    --ui-font-family: 'Nunito', sans-serif;
  }
}
```

## Importing External Token Systems

### W3C Design Tokens Format (DTCG)

```json
{
  "color": {
    "primary": { "$value": "#0ea5e9", "$type": "color" },
    "bg":      { "$value": "#f0f9ff", "$type": "color" }
  },
  "radius": {
    "md": { "$value": "10px", "$type": "dimension" }
  }
}
```

```bash
node scripts/theme-map.js ./w3c-tokens.json -o ./dist/themes/my-theme.css --name "My Theme"
```

### Figma Tokens Plugin format

```json
{
  "color": {
    "primary": { "value": "#0ea5e9", "type": "color" },
    "bg":      { "value": "#f0f9ff", "type": "color" }
  }
}
```

The mapper auto-detects the format. For W3C and Figma inputs, the nested key
path is flattened to dot-notation and then mapped to CSS variables using the
same rules as the native format.

> **Note:** For Figma or W3C tokens with custom organisational structure, the
> flattened key must match the expected dot-notation pattern. If your Figma
> file uses `global.colors.primary` instead of `color.primary`, the output
> will be `--ui-global-colors-primary`. Use the native format or rename keys
> to match the ai-css-kit convention.

## How `@layer` Enables Override-Free Theming

Without `@layer`, overriding a component style requires matching or beating its
specificity. With layers:

```css
/* Component default — in ai-kit.components */
.ui-button--primary { background: var(--ai-button-bg); }

/* Theme override — in ai-kit.themes (declared later = higher priority) */
[data-theme="ocean"] { --ui-color-primary: #0ea5e9; }
```

Because `ai-kit.themes` is declared after `ai-kit.components` in the layer
order, any rule in a theme automatically wins — even a zero-specificity `:root`
rule beats a high-specificity component rule from an earlier layer.

## Building All Preset Themes

```bash
npm run build:themes
# Reads src/themes/*.json and writes dist/themes/*.css
```

This is included in the full `npm run build` pipeline.

---

## npm Exports

Theme CSS files are exposed via the `./themes/*` export map:

```js
// In a bundler that supports CSS exports:
import 'ai-css-kit/themes/midnight';
```

Or directly in HTML:

```html
<link rel="stylesheet" href="node_modules/ai-css-kit/dist/themes/midnight.css" />
```
