# AI CSS Kit — Glossary / Глоссарий

> This file is the single source of glossary for all languages.  
> Этот файл — единый глоссарий для всех языков.

The table lists every term used across the project documentation.  
Таблица содержит все термины, используемые в документации проекта.

| Term (EN) | Термин (RU) | English definition | Определение на русском |
|-----------|-------------|-------------------|------------------------|
| **ADR** (Architectural Decision Record) | **Запись архитектурного решения** | A short document capturing a specific architectural decision: what was decided, why, and which alternatives were rejected. Stored in `docs/adr/`. | Короткий документ, фиксирующий конкретное архитектурное решение: что решено, почему и какие альтернативы были отклонены. Хранится в `docs/adr/`. |
| **Atom** | **Атом** | The smallest indivisible UI unit — a single component with one file and one responsibility. Cannot be further decomposed without losing meaning. | Наименьшая неделимая единица UI — один компонент, один файл, одна ответственность. Не может быть декомпозирована без потери смысла. |
| **Base class** | **Базовый класс** | The primary CSS class that applies foundational styles to a component (e.g., `.ui-input`, `.ui-button`). Must always be present before modifiers. | Основной CSS-класс, применяющий базовые стили к компоненту (напр., `.ui-input`, `.ui-button`). Должен присутствовать всегда — до модификаторов. |
| **Bundle** | **Бандл / Сборка** | The compiled CSS output, typically `dist/ai-css-kit.css` or its minified variant. Produced by the PostCSS build pipeline. | Скомпилированный CSS-вывод, обычно `dist/ai-css-kit.css` или его минифицированный вариант. Создаётся пайплайном PostCSS. |
| **Cascade layer** (`@layer`) | **Каскадный слой** | A CSS feature that groups rules into named layers. Layer declared later always wins over an earlier one, regardless of specificity. Used for the `ai-kit.tokens → ai-kit.base → ai-kit.components → ai-kit.themes` order. | Возможность CSS, группирующая правила в именованные слои. Слой, объявленный позже, всегда побеждает более ранний независимо от специфичности. Используется для порядка `ai-kit.tokens → ai-kit.base → ai-kit.components → ai-kit.themes`. |
| **Component token** | **Токен компонента** | A CSS custom property prefixed `--ai-[component]-*` (Level 2). Scoped to a single component. References a global token by default and can be overridden without cascade side effects. | CSS-переменная с префиксом `--ai-[component]-*` (Уровень 2). Применяется к одному компоненту. По умолчанию ссылается на глобальный токен; может быть переопределена без побочных каскадных эффектов. |
| **Context modifier** | **Контекстный модификатор** | An explicit CSS class added directly to an atom to describe the context in which it is used (e.g., `.ui-input--icon-right`). Replaces parent-selector mutation. The name answers "in what context?" not "who is using it?". | Явный CSS-класс, добавляемый непосредственно к атому для описания контекста использования (напр., `.ui-input--icon-right`). Заменяет мутацию через parent-selector. Имя отвечает на вопрос «в каком контексте?», а не «кто использует?». |
| **Coupling** | **Зацепление (связность)** | An unwanted dependency between components where one component's CSS affects the styles of another. Detected and prohibited by `scripts/audit-coupling.js`. | Нежелательная зависимость между компонентами, когда CSS одного компонента влияет на стили другого. Обнаруживается и запрещается скриптом `scripts/audit-coupling.js`. |
| **CSS custom property** | **CSS-переменная / Кастомное свойство CSS** | A variable declared with two leading dashes (`--variable-name`) and accessed via `var(--variable-name)`. Used for all design tokens and component customisation points. | Переменная, объявляемая с двумя тире (`--variable-name`) и доступная через `var(--variable-name)`. Используется для всех дизайн-токенов и точек кастомизации компонентов. |
| **Design token** | **Дизайн-токен** | A named value that stores a design decision (colour, spacing, radius, shadow, etc.) as a CSS custom property. Enables consistent theming across all components. | Именованное значение, хранящее решение в области дизайна (цвет, отступ, радиус, тень и т.д.) в виде CSS-переменной. Обеспечивает единообразную темизацию всех компонентов. |
| **Deterministic UI** | **Детерминированный UI** | A UI in which identical inputs (same HTML structure + same tokens) always produce identical visual output, with no hidden side effects or context-dependent behaviour. Core principle of ai-css-kit. | UI, в котором одинаковые входные данные (одна и та же HTML-структура + одни и те же токены) всегда дают одинаковый визуальный результат без скрытых побочных эффектов. Ключевой принцип ai-css-kit. |
| **Deterministic UI Lego** | **Детерминированное UI-Lego** | The project's central concept: components are atomic, isolated, and self-documenting — like Lego bricks — so any combination is predictable and machine-composable without ambiguity. | Центральная концепция проекта: компоненты атомарны, изолированы и самодокументируемы — как кубики Lego — поэтому любая их комбинация предсказуема и может быть составлена машиной без двусмысленности. |
| **Dragover state** | **Состояние dragover** | The visual state of a file upload zone when a file is dragged over it. Applied via `.ui-file-upload--dragover` class (set by JavaScript). | Визуальное состояние зоны загрузки файла, когда над ней перетаскивается файл. Применяется через класс `.ui-file-upload--dragover` (устанавливается JavaScript). |
| **Field wrapper** | **Обёртка поля** | The `.ui-field` container component that provides consistent spacing and size context for form elements. Every interactive form atom must be placed inside a field wrapper with a size modifier (`--sm`, `--md`, `--lg`). | Компонент-контейнер `.ui-field`, обеспечивающий единообразные отступы и контекст размера для элементов формы. Каждый интерактивный атом формы должен быть помещён в обёртку поля с модификатором размера (`--sm`, `--md`, `--lg`). |
| **Figma Tokens Plugin format** | **Формат Figma Tokens Plugin** | A JSON token format used by the Figma Tokens Plugin, where values are stored as `{ "value": "...", "type": "..." }`. Supported by the Theme Mapper CLI. | Формат JSON-токенов, используемый плагином Figma Tokens, где значения хранятся как `{ "value": "...", "type": "..." }`. Поддерживается Theme Mapper CLI. |
| **Global token** | **Глобальный токен** | A CSS custom property prefixed `--ui-*` (Level 1). Affects all components that reference it. Used for brand-wide changes. | CSS-переменная с префиксом `--ui-*` (Уровень 1). Влияет на все компоненты, которые ссылаются на неё. Используется для общебрендовых изменений. |
| **Module Contract** | **Контракт модуля** | The specification that every component must meet: all states via own modifier classes, no CSS targeting foreign classes, order-of-import independence, correct rendering in isolation, full HTML documentation in SPEC.md. | Спецификация, которой должен соответствовать каждый компонент: все состояния через собственные классы-модификаторы, отсутствие CSS-правил, таргетирующих чужие классы, независимость от порядка импорта, корректный рендер в изоляции, полная HTML-документация в SPEC.md. |
| **Orchestrator** | **Оркестратор** | A composite component that coordinates multiple atoms without mutating their internal styles (e.g., Datepicker, Autocomplete). Uses context modifiers to configure child atoms explicitly. | Составной компонент, координирующий несколько атомов без мутации их внутренних стилей (напр., Datepicker, Autocomplete). Использует контекстные модификаторы для явной настройки дочерних атомов. |
| **Parent-selector mutation** | **Мутация через родительский селектор** | The anti-pattern of overriding a child component's styles from a parent's CSS selector (e.g., `.ui-datepicker .ui-input { padding-right: 40px }`). Prohibited by ADR-0001. | Антипаттерн переопределения стилей дочернего компонента из CSS-селектора родителя (напр., `.ui-datepicker .ui-input { padding-right: 40px }`). Запрещён ADR-0001. |
| **PostCSS** | **PostCSS** | A tool for transforming CSS with JavaScript plugins. Used in ai-css-kit for `@import` resolution, CSS nesting syntax, autoprefixing, and minification. | Инструмент для преобразования CSS с помощью JavaScript-плагинов. Используется в ai-css-kit для раскрытия `@import`, синтаксиса вложенности CSS, автопрефиксинга и минификации. |
| **Preset theme** | **Пресет темы** | A pre-built CSS file in `dist/themes/` that can be activated by setting `data-theme="{name}"` on `<html>`. Five presets ship with the kit: `default`, `dark`, `midnight`, `corporate`, `warm`. | Готовый CSS-файл в `dist/themes/`, который активируется установкой `data-theme="{name}"` на `<html>`. В кит входят пять пресетов: `default`, `dark`, `midnight`, `corporate`, `warm`. |
| **Size modifier** | **Модификатор размера** | A class suffix (`--sm`, `--md`, `--lg`) that controls the visual dimensions of a component or field. On `.ui-field`, it propagates sizing to all child form atoms automatically. | Суффикс класса (`--sm`, `--md`, `--lg`), управляющий визуальными размерами компонента или поля. На `.ui-field` автоматически распространяет размеры на все дочерние атомы формы. |
| **State modifier** | **Модификатор состояния** | A class suffix that represents a validation or interaction state (e.g., `--error`, `--success`, `--warning`, `--dragover`). Applied to the component itself, not to its parent. | Суффикс класса, представляющий состояние валидации или взаимодействия (напр., `--error`, `--success`, `--warning`, `--dragover`). Применяется к самому компоненту, а не к его родителю. |
| **Theme Mapper** | **Маппер тем** | The CLI tool at `scripts/theme-map.js` that converts JSON design tokens (native, W3C DTCG, or Figma format) into a ready-to-use CSS theme file wrapped in `@layer ai-kit.themes`. | CLI-инструмент в `scripts/theme-map.js`, конвертирующий JSON-токены дизайна (родной формат, W3C DTCG или Figma) в готовый CSS-файл темы, обёрнутый в `@layer ai-kit.themes`. |
| **Token namespace** | **Пространство имён токенов** | The prefix that groups related CSS custom properties. Level 1: `--ui-*` (global). Level 2: `--ai-[component]-*` (component-scoped). | Префикс, группирующий связанные CSS-переменные. Уровень 1: `--ui-*` (глобальный). Уровень 2: `--ai-[component]-*` (на уровне компонента). |
| **Tree-shaking** | **Тришейкинг (удаление мёртвого кода)** | A bundler optimisation that removes unused code. Supported via subpath exports: `@import 'ai-css-kit/components/button.css'` imports only the button component, not the full bundle. | Оптимизация бандлера, удаляющая неиспользуемый код. Поддерживается через subpath-экспорты: `@import 'ai-css-kit/components/button.css'` импортирует только компонент кнопки, а не весь бандл. |
| **Validation state** | **Состояние валидации** | One of three contextual feedback states applied to a form input: `--error` (red), `--success` (green), `--warning` (amber). Always set on the `<input>` element itself, not on `.ui-field`. | Одно из трёх контекстных состояний обратной связи, применяемых к полю ввода: `--error` (красный), `--success` (зелёный), `--warning` (жёлтый). Всегда задаётся на элементе `<input>`, а не на `.ui-field`. |
| **W3C Design Tokens Community Group format (DTCG)** | **Формат W3C Design Tokens Community Group (DTCG)** | A standardised JSON token format where values use `{ "$value": "...", "$type": "..." }`. Supported by the Theme Mapper CLI for cross-tool compatibility. | Стандартизированный JSON-формат токенов, где значения используют `{ "$value": "...", "$type": "..." }`. Поддерживается Theme Mapper CLI для совместимости между инструментами. |
| **Zero dependencies** | **Без зависимостей (Zero deps)** | The runtime principle: ai-css-kit ships only CSS. No JavaScript framework or runtime library is required. Font Awesome is the only external CDN dependency (for icons). | Принцип рантайма: ai-css-kit поставляется только в виде CSS. Не требует JavaScript-фреймворка или библиотеки во время выполнения. Font Awesome — единственная внешняя CDN-зависимость (для иконок). |

## Adding New Terms / Добавление новых терминов

To add a term, insert a new row in the table above with both English and Russian columns filled.  
Чтобы добавить термин, вставьте новую строку в таблицу выше, заполнив оба столбца.

If adding a term in a new language, add a column to the table header and all existing rows.  
Чтобы добавить термин на новом языке, добавьте столбец в заголовок таблицы и во все существующие строки.
