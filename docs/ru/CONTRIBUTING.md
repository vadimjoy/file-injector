# Участие в разработке ai-css-kit

> **Язык:** Русский | [English](../en/CONTRIBUTING.md)

> **Примечание:** `ai-css-kit` — каноническое название в коде, коммитах, PR и документации.

---

## Видение

Мы создаём не UI-кит. Мы создаём **Lego для нейросетей**.

Каждый компонент `ai-css-kit` должен быть:

- **Детерминированным** — для одной и той же HTML-структуры и набора токенов любой AI-агент должен выдавать идентичный, предсказуемый результат
- **Атомарным** — один блок, один файл, одна ответственность. Никаких скрытых связей
- **Самодостаточным** — Calendar не является дочерним элементом Input. Dropdown не принадлежит Autocomplete. Каждый — независимая CSS-сущность

Конечная цель: AI-агент должен уметь скомпоновать любой UI из этого кита без двусмысленности, без магии и без чтения исходного кода.

---

## Архитектурные принципы

### 1. Pure CSS и декаплинг

**Правило:** Один компонент = один файл = одна задача.

Компоненты, совмещающие состояние (например, всплывающий календарь, встроенный в DOM поля ввода), — это архитектурный долг. Правильная модель:

```
src/css/components/
├── input.css          # только текстовый ввод
├── calendar.css       # сетка календаря — самостоятельная CSS-сущность
├── autocomplete.css   # список предложений — самостоятельная CSS-сущность
├── dropdown.css       # универсальная оболочка выпадающего списка
└── datepicker.css     # композиция: input + calendar (через Anchor Positioning)
```

`datepicker.css` может *ссылаться* на токены `calendar.css`, но `calendar.css` должен корректно рендериться в изоляции.

**CSS в первую очередь, JS — в крайнем случае.** Прежде чем прибегать к JavaScript:

| Задача | CSS-решение | JS-запасной вариант (последнее средство) |
|--------|-------------|------------------------------------------|
| Позиционирование попапа | `anchor-positioning` | `getBoundingClientRect` |
| Условный макет | `container-queries` | resize observer |
| Переключение состояний | `:has()`, checkbox hack | переключение классов |
| Прокрутка с привязкой | `scroll-snap-type` | JS scroll listener |

Если CSS-решение существует в современных браузерах (Chrome 90+, Firefox 88+, Safari 14+) — используйте его.

### 2. Соглашение по именованию токенов

Все CSS-кастомные свойства следуют строгой схеме:

```
--ai-[component]-[property]
```

Примеры:

```css
/* tokens.css — глобальные примитивы */
--ai-color-primary: #4f46e5;
--ai-space-4: 16px;
--ai-radius-md: 8px;
--ai-shadow-md: 0 4px 6px rgba(0, 0, 0, 0.07);

/* токены уровня компонента */
--ai-input-height-md: 40px;
--ai-input-border-color: var(--ai-color-border);
--ai-calendar-day-size: 36px;
--ai-calendar-header-gap: var(--ai-space-2);
--ai-button-font-weight: 500;
```

**Никаких магических чисел.** Каждое захардкоженное значение — это баг, ждущий своего часа. Если число появляется в CSS компонента, оно должно ссылаться на токен:

```css
/* НЕПРАВИЛЬНО */
.ui-input { padding: 9px 13px; border-radius: 8px; }

/* ПРАВИЛЬНО */
.ai-input { padding: var(--ai-input-padding-y) var(--ai-input-padding-x); border-radius: var(--ai-radius-md); }
```

### 3. Совместимость с темами

Кит должен уметь «поглощать» любую внешнюю JSON-тему. Токены тем отображаются 1:1 на CSS-переменные:

```json
{
  "ai-color-primary": "#10b981",
  "ai-radius-md": "12px",
  "ai-font-family": "\"Poppins\", sans-serif"
}
```

```javascript
// применение темы (паттерн demo-theme.js)
Object.entries(theme).forEach(([key, value]) => {
  document.documentElement.style.setProperty(`--${key}`, value);
});
```

Любой компонент, ломающийся при произвольной теме, — это дефект.

## Процесс участия в разработке

### Перед началом работы

- Проверьте [открытые задачи](https://github.com/vadimjoy/ai-css-kit/issues) и [открытые PR](https://github.com/vadimjoy/ai-css-kit/pulls), чтобы избежать дублирования
- Для новых компонентов или значительных изменений сначала откройте задачу для согласования объёма работ

### Для крупных рефакторингов — напишите ADR

Любое структурное изменение (например, декаплинг Calendar от Input, переход на Anchor Positioning, переименование схемы токенов) требует **Architectural Decision Record** перед реализацией.

Создайте файл по пути:

```
docs/adr/NNNN-short-title.md
```

Шаблон ADR:

```markdown
# ADR-NNNN: [Название]

**Статус:** Proposed | Accepted | Deprecated

## Контекст
Какую проблему решаем и почему сейчас?

## Решение
Что именно делаем?

## Последствия
- Что сломается?
- Что улучшится?
- Путь миграции для потребителей?
```

Примеры изменений, требующих ADR:
- Декаплинг `datepicker` → `input` + `calendar`
- Миграция пространства имён токенов с `--ui-*` на `--ai-*`
- Добавление Anchor Positioning как стратегии позиционирования попапов по умолчанию
- Введение новой категории компонентов

### Настройка окружения

```bash
git clone https://github.com/vadimjoy/ai-css-kit.git
cd ai-css-kit
npm install
npm run build
```

Откройте `index.html` в браузере для просмотра витрины компонентов.

### Рабочий процесс разработки

1. Создайте ветку функциональности от `main`:
   ```bash
   git checkout -b feat/calendar-decoupling
   ```

2. Работайте в `src/css/components/`. Каждый компонент — отдельный `.css`-файл с вложенностью PostCSS.

3. Добавьте или обновите соответствующее демо в `src/demos/`.

4. Соберите и проверьте:
   ```bash
   npm run build
   ```

5. Проверьте вручную темы `data-theme="light"` и `data-theme="dark"`.

### Соглашение о коммитах

Следуйте [Conventional Commits](https://www.conventionalcommits.org/):

```
feat(calendar): add standalone calendar component
fix(input): remove hardcoded border-radius magic number
refactor(tokens): migrate --ui-* to --ai-* namespace
docs(adr): add ADR-0001 for calendar decoupling
```

**Запрещено в сообщениях коммитов и описаниях PR:** любые устаревшие кодовые названия. Используйте `ai-css-kit`.

### Требования к Pull Request

- [ ] Нет магических чисел — все значения ссылаются на токены
- [ ] Имена переменных следуют схеме `--ai-[component]-[property]`
- [ ] Компонент корректно рендерится в изоляции (нет скрытых зависимостей от родителя)
- [ ] Проверены обе темы: светлая и тёмная
- [ ] Для нового компонента: добавлен demo-файл в `src/demos/`
- [ ] Для ломающего изменения или крупного рефакторинга: ADR существует в `docs/adr/`
- [ ] Ноль ссылок на устаревшие кодовые названия в изменённых файлах

## Стиль кода

- **Именование селекторов:** BEM-стиль с префиксом `ui-`: `.ui-input`, `.ui-input__icon`, `.ui-input--error`
- **Вложенность:** Используйте вложенность PostCSS (`postcss-nesting`). Максимум 3 уровня
- **Комментарии:** Обозначайте секции компонента блочным заголовком:
  ```css
  /* ─── Calendar Header ─── */
  ```
- **Единицы:** `px` для фиксированной геометрии, `em`/`rem` для значений, относительных к типографике, `%` для fluid-макета
