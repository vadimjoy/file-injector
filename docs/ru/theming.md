> **Язык:** Русский | [English](../en/theming.md)

> **Версия:** v0.6.0  
> **Связанный ADR:** [ADR-0003 Нейминг AI-токенов](adr/0003-ai-token-naming.md)

---

## Обзор

AI CSS Kit использует двухуровневую архитектуру токенов и CSS `@layer`, чтобы сделать темизацию детерминированной: файл темы всегда побеждает стили компонентов по умолчанию без `!important`.

```
@layer ai-kit.tokens    ← Глобальные --ui-* значения по умолчанию + оверрайд тёмной темы
@layer ai-kit.base      ← Reset, .ui-field, .ui-label, …
@layer ai-kit.components ← Стили всех 15 компонентов (--ai-* токены)
@layer ai-kit.themes    ← Пресеты тем / кастомные темы (наивысший приоритет)
```

Любое правило, помещённое в `ai-kit.themes`, автоматически имеет приоритет над всеми стилями компонентов, независимо от специфичности CSS.

## Использование пресета темы

В `dist/themes/` включены пять пресетов:

| Файл | Описание |
|------|----------|
| `dist/themes/default.css` | Светлая тема (совпадает со встроенными значениями `:root`) |
| `dist/themes/dark.css` | Тёмная тема (совпадает со встроенным `[data-theme="dark"]`) |
| `dist/themes/midnight.css` | Глубокий тёмный с фиолетовым акцентом, моноширинный шрифт |
| `dist/themes/corporate.css` | Нейтральный синий, острые углы, системный sans-serif |
| `dist/themes/warm.css` | Тёплые янтарные оттенки, щедрые скругления, шрифт с засечками |

### 1. Подключите тему после основного бандла

```html
<link rel="stylesheet" href="dist/ai-css-kit.css" />
<link rel="stylesheet" href="dist/themes/midnight.css" />
```

### 2. Активируйте через атрибут `data-theme`

```html
<html data-theme="midnight">
  ...
</html>
```

Переключение тем во время выполнения — одна строка JavaScript:

```js
document.documentElement.setAttribute('data-theme', 'corporate');
```

## Создание кастомной темы

### Родной формат (рекомендуется)

Создайте JSON-файл с блоком `meta` и плоским объектом `tokens`:

```json
{
  "meta": {
    "name": "Ocean Pro",
    "version": "1.0.0",
    "author": "designer@example.com"
  },
  "tokens": {
    "color.primary":       "#0ea5e9",
    "color.primary.hover": "#0284c7",
    "color.bg":            "#f0f9ff",
    "radius.md":           "10px",
    "font.family":         "'Nunito', sans-serif"
  }
}
```

### Маппинг ключей токенов на CSS-переменные

| Паттерн ключа токена | CSS-переменная | Пример |
|----------------------|----------------|--------|
| `color.*`            | `--ui-color-*` | `color.primary` → `--ui-color-primary` |
| `radius.*`           | `--ui-radius-*` | `radius.md` → `--ui-radius-md` |
| `font.*`             | `--ui-font-*` | `font.family` → `--ui-font-family` |
| `shadow.*`           | `--ui-shadow-*` | `shadow.sm` → `--ui-shadow-sm` |
| `button.*`           | `--ai-button-*` | `button.bg` → `--ai-button-bg` |
| `input.*`            | `--ai-input-*` | `input.border` → `--ai-input-border` |
| `card.*`             | `--ai-card-*` | `card.radius` → `--ai-card-radius` |
| `--custom-var`       | дословно | `--ui-line-height` → `--ui-line-height` |

Имена компонентов, отображаемых на `--ai-*`: `button`, `input`, `textarea`, `select`,
`card`, `badge`, `progress`, `tooltip`, `checkbox`, `radio`, `toggle`,
`slider`, `file-upload`, `autocomplete`, `calendar`.

Все остальные ключи токенов отображаются на `--ui-*`.

### Генерация CSS

```bash
node scripts/theme-map.js ./my-theme.json -o ./dist/themes/ocean-pro.css
```

Или через алиас npm-скрипта:

```bash
npm run theme-map -- ./my-theme.json -o ./dist/themes/ocean-pro.css
```

Сгенерированный CSS:

```css
/* Generated by ai-css-kit theme-map — do not edit */
/* Theme:   Ocean Pro */
/* Version: 1.0.0 */

@layer ai-kit.themes {
  [data-theme="ocean-pro"] {
    --ui-color-primary: #0ea5e9;
    --ui-color-primary-hover: #0284c7;
    --ui-color-bg: #f0f9ff;
    --ui-radius-md: 10px;
    --ui-font-family: 'Nunito', sans-serif;
  }
}
```

## Импорт внешних систем токенов

### Формат W3C Design Tokens (DTCG)

```json
{
  "color": {
    "primary": { "$value": "#0ea5e9", "$type": "color" },
    "bg":      { "$value": "#f0f9ff", "$type": "color" }
  },
  "radius": {
    "md": { "$value": "10px", "$type": "dimension" }
  }
}
```

```bash
node scripts/theme-map.js ./w3c-tokens.json -o ./dist/themes/my-theme.css --name "My Theme"
```

### Формат Figma Tokens Plugin

```json
{
  "color": {
    "primary": { "value": "#0ea5e9", "type": "color" },
    "bg":      { "value": "#f0f9ff", "type": "color" }
  }
}
```

Маппер автоматически определяет формат. Для входных данных W3C и Figma вложенный путь ключа разворачивается в dot-notation, а затем отображается на CSS-переменные по тем же правилам, что и родной формат.

> **Примечание:** Для токенов Figma или W3C с пользовательской организационной структурой сплющенный ключ должен соответствовать ожидаемому паттерну dot-notation. Если ваш файл Figma использует `global.colors.primary` вместо `color.primary`, результатом будет `--ui-global-colors-primary`. Используйте родной формат или переименуйте ключи в соответствии с соглашением ai-css-kit.

## Как `@layer` обеспечивает темизацию без перезаписей

Без `@layer` для переопределения стиля компонента нужно сопоставить или превзойти его специфичность. С каскадными слоями:

```css
/* Стиль компонента по умолчанию — в ai-kit.components */
.ui-button--primary { background: var(--ai-button-bg); }

/* Переопределение темой — в ai-kit.themes (объявлено позже = выше приоритет) */
[data-theme="ocean"] { --ui-color-primary: #0ea5e9; }
```

Поскольку `ai-kit.themes` объявлен после `ai-kit.components` в порядке слоёв, любое правило в теме автоматически побеждает — даже правило `:root` с нулевой специфичностью побеждает высокоспецифичное правило компонента из более раннего слоя.

## Сборка всех пресетных тем

```bash
npm run build:themes
# Читает src/themes/*.json и записывает dist/themes/*.css
```

Это включено в полный пайплайн `npm run build`.

## npm-экспорты

CSS-файлы тем доступны через карту экспортов `./themes/*`:

```js
// В бандлере с поддержкой CSS-экспортов:
import 'ai-css-kit/themes/midnight';
```

Или напрямую в HTML:

```html
<link rel="stylesheet" href="node_modules/ai-css-kit/dist/themes/midnight.css" />
```
